# Premise Distillation Protocol

Use during init and decompose-plan when the user intent is broad.

Create a private goal statement with:

- what changes when this is done
- why now
- success criteria
- explicit non-goals
- validated premises
- open premises
- corrected premises

The goal statement is an anchor, not a rigid gate. If premises are unresolved,
mark them draft and surface them as checks during decomposition.

Do not block implementation on perfect premise capture when the user clearly
wants momentum.
